Модуль 1: Введение в ОС Astra Linux

Данный модуль является первым в курсе о базовом администрировании операционной системы Astra Linux Special Edition 1.8 с использованием графических инструментов. Здесь вы познакомитесь с основными событиями в развитии ОС, хронологией выхода её версий, а также с информацией, которую обозначает номер версии Astra Linux.

История Astra Linux

Astra Linux — операционная система специального назначения на базе ядра Linux, созданная для нужд органов государственного и военного управления и других учреждений, которые работают с информацией ограниченного доступа.

Разработка начата в 2008 году АО «НПО РусБИТех» при участии Министерства обороны РФ. Встроенные средства защиты ОС разработаны совместно с Академией ФСБ России и Институтом системного программирования РАН.

Дистрибутивы Astra Linux

Дистрибутив Linux — это комплект взаимосогласованного программного обеспечения, включающий ОС на базе ядра Linux, системное ПО и приложения, а также систему управления программным обеспечением.

Дистрибутивы семейства Astra Linux являются официальными производными дистрибутива Debian и могут иметь несколько вариантов исполнения, отличающихся назначением, версией репозитория и целевой архитектурой.

Уровни защищённости ОС Astra Linux

Впервые в российской практике в одном релизе Astra Linux Special Edition реализованы несколько уровней защиты, включая безопасную среду виртуализации.

  • Усиленный уровень («Воронеж») — рекомендуется для обработки конфиденциальной информации в государственных информационных системах.
  • Максимальный уровень («Смоленск») — рекомендуется для обработки информации любой категории доступа, включая сведения, составляющие гостайну.
  • Допустимый уровень — определяется приобретённой лицензией на использование ОС.

Модуль 3: Процесс загрузки ОС

Данный модуль содержит основные сведения о стадиях загрузки ОС. Вы ознакомитесь с загрузчиком ОС, его функциями и основными возможностями настройки.

Порядок загрузки ОС

  1. При подаче питания процессор выполняет код BIOS/UEFI.
  2. Загружается программа-загрузчик (GRUB2) из MBR или EFI-раздела.
  3. GRUB2 загружает ядро Linux и образ initramfs в оперативную память.
  4. Ядро инициализирует важные функции, монтирует initramfs и запускает процесс init.
  5. После загрузки драйверов монтируется системный раздел, процесс init замещается службой systemd.
  6. Systemd запускает службы, виртуальные терминалы и графическую подсистему.

Загрузчик GRUB2

GRUB2 — многофункциональный загрузчик, используемый в современных системах семейства Linux.

  • Отображает варианты загрузки и позволяет редактировать параметры загрузки.
  • Поддерживает загрузку нескольких операционных систем.
  • Имеет модульную структуру, позволяет загружать драйверы для распознавания файловых систем.
  • Загружает ядро Linux и образ initramfs, передаёт управление ядру.

Systemd — система инициализации

Systemd — это комплекс программ, экосистема которых формируется вокруг одноимённой службы. Является «корневым» родителем всех процессов пользовательского пространства.

Основные компоненты systemd:

  • systemd-journald — служба регистрации событий.
  • systemd-networkd — служба управления сетевыми соединениями.
  • systemd-logind — служба управления входом пользователей.
  • systemd-udevd — служба управления событиями устройств.
